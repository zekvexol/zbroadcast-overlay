Chat Title: Manual Esports Overlay #1
1. Current App State & Progress
The project is a working local prototype of a custom esports broadcast scoreboard web app intended to replace Rocket League community casting tools like BARL/CARL after the removal of third-party injection support. The system is currently built as a Node.js + Express + Socket.IO app with a browser-based control panel and a browser-based overlay page suitable for use in OBS Browser Source.
What is currently built and working:

* A local Node.js project folder exists and runs successfully.

* server.js is running and serves static pages from a public folder.

* Real-time communication is working through Socket.IO.

* A control page exists at http://localhost:3000/control.html.

* An overlay page exists at http://localhost:3000/overlay.html.

* OBS can load the overlay page as a Browser Source and it displays correctly.

* Score updates are instant and feel much faster than the original Google Sheets + Apps Script idea.

* The system tracks:

  * blueName

  * orangeName

  * blueScore

  * orangeScore

  * blueSeries

  * orangeSeries

* Team naming has been standardized to Blue and Orange instead of Team A / Team B, because that matches both Rocket League and Rainbow Six spectator conventions.

* A reset structure exists with three levels:

  * Reset Game: clears only game score

  * Reset Series: clears only series score

  * Full Reset (New Match): resets names to Blue/Orange, game score, and series score

* Undo logic exists:

  * Undo Blue Goal

  * Undo Orange Goal

  * scores cannot go below 0

* A fully functioning hotkey system exists:

  * configurable in the control panel

  * saved to browser localStorage

  * hotkeys do not trigger while typing in text fields

* The control panel has been heavily refined into an operator-friendly layout:

  * Blue panel on left

  * Orange panel on right

  * each team has a goal control row with:

    * info label

    * vertically stacked + and -

    * live current score display box

  * match control buttons below

  * floating top-right hotkey gear menu

* The overlay has been updated from a basic development text bar to a more broadcast-style layout:

  * Blue panel on left

  * center timer cutout area

  * Orange panel on right

  * series pips displayed visually

  * Blue pips fill left to right

  * Orange pips fill right to left

What is partially done:

* A custom overlay image upload system was started.

* The control panel already has:

  * Upload Overlay Image

  * Clear Overlay Image

* The overlay page already contains logic to try to use an uploaded image as a full 1920x1080 background.

* However, this first version used browser localStorage to store the uploaded image.

* That localStorage-based overlay image workflow works inside the same browser environment, but does not work correctly in OBS Browser Source, because OBS does not reliably share the same browser storage context as the normal browser tab.

What is still needed immediately:

* Convert overlay image upload from browser localStorage storage to server-side storage using a Node upload middleware such as Multer.

* Save uploaded overlay images into a server-readable folder, likely under public/overlay-assets/.

* Serve that file from a stable URL such as /overlay-assets/overlay.png.

* Update overlay.html to load the server-stored overlay image instead of reading from localStorage.

* Update control.html upload logic to send the file to the server rather than using FileReader + localStorage.

* Ensure OBS and all clients load the same overlay asset from the server path.

Larger future direction:

* Move from local-only prototype to remote-capable multi-device use.

* Eventually allow score operators on other PCs and other networks to control the scoreboard.

* Later phases include multi-room support and potential OCR/timer experimentation.
